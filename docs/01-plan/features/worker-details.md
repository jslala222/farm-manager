# PDCA 계획서: 근로자 상세 프로필(성별, 주소, 메모) 확장 및 DB 복구

## 1. 개요 (Plan - P)
사용자 요청에 따라 모든 근로자(사장, 가족, 직원, 알바)의 인적 사항을 보다 상세히 관리할 수 있도록 성별, 주소, 메모 필드를 추가합니다. 또한, 현재 'phone' 컬럼 누락으로 발생한 등록 오류를 즉시 해결하여 시스템 안정성을 확보합니다.

## 2. 현황 및 문제점
- **문제 1**: 근로자 관리 시 '연락처(phone)' 컬럼이 DB에 없어 등록/수정 시 에러 발생. (에러 확인 완료)
- **문제 2**: 현재 이름과 역할만 관리 가능하며, 실제 농장 운영에 필요한 성별, 주소, 메모 등의 정보 기록 불가.

## 3. 목표
- DB 테이블(`workers`)에 `phone`, `gender`, `address`, `notes` 컬럼 추가.
- 근로자 등록 및 수정 UI에 성별(남/여) 선택, 주소 입력, 장문 메모 입력 기능 통합.
- 사장님을 포함한 모든 인력의 DB 관리 체계 정비.

## 4. 작업 계획
1. **분석**: 현재 `workers` 테이블 스키마 확인 및 컬럼 누락 원인 파악 (완료).
2. **설계**: 상세 정보를 포함한 신규 UI 레이아웃 설계 (D단계).
3. **구현 (Do)**:
    - **DB 업데이트**: 사용자에게 필수 SQL 제공 (스키마 복구 및 확장).
    - **프론트엔드 수정**: `app/workers/page.tsx`에 신규 필드 UI 및 데이터 연동 작업.
4. **검증 (Check)**: 갭 분석 및 실제 등록/수정 테스트.
5. **보고 (Act)**: 완료 보고서 작성 및 사용자 가이드 제공.
