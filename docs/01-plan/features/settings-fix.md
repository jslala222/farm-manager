# PDCA 계획서: 설정 페이지 안정화 및 버그 수정

## 1. 개요 (Plan - P)
현재 '설정' 페이지에서 관리자 계정으로 테스트 농장을 등록하려고 할 때, [농장 등록하기] 버튼이 반응하지 않는 문제가 보고되었습니다. 또한, 초기 농장 등록 시 동 개수 설정 기능의 안정성을 확보해야 합니다.

## 2. 현황 및 문제점
- **문제**: 설정 페이지의 [농장 등록하기] 버튼 클릭 시 아무런 반응(Alert, Loading 등)이 없음.
- **원인 가설**:
    1. `useAuthStore`의 `initialized` 상태가 `false`로 유지되어 컴포넌트가 정상적으로 렌더링되지 않거나 이벤트가 바인딩되지 않음.
    2. 버튼 클릭 이벤트 핸들러(`handleSaveFarm`) 내에서 예외가 발생하여 중단됨.
    3. 모바일 뷰(스마트폰 사이즈)에서 UI 요소가 겹치거나 이벤트가 차단될 가능성.

## 3. 목표
- [농장 등록하기] 및 [정보 수정하기] 버튼의 100% 정상 작동 보장.
- 농장 등록 시 입력한 '하우스 동 개수'만큼 자동으로 `farm_houses` 데이터 생성 확인.
- bkit 규정에 따른 PDCA 사이클 완수 (Plan -> Do -> Check -> Act).

## 4. 작업 계획
1. **분석**: `handleSaveFarm` 로직에 상세 로그(`console.log`)를 추가하여 어느 단계에서 멈추는지 확인.
2. **수정**: 안정적인 Auth 세션 감지 로직 적용 및 버튼 가시성/작동성 개선.
3. **검증**: 직접 테스트 스크립트 또는 UI 테스트를 통해 등록 프로세스 확인.
4. **보고**: 완료 보고서 작성 및 사용자 가이드 제공.
