# PDCA 계획서: 근로자 관리 CRUD 고도화 및 출근 중복 방지

## 1. 개해 (Plan - P)
사용자 피드백에 따라 근로자 관리 기능(수정/추가/삭제)을 강화하고, 출근 기록이 중복으로 노출되거나 저장되는 현상을 해결하여 데이터 정합성을 확보합니다.

## 2. 현황 및 문제점
- **근로자 관리**: 현재 '수정(Update)' 기능이 누락되어 인적 사항 변경 시 삭제 후 재등록해야 함. 또한 알바/식구 등 카테고리별 관리가 직관적이지 않음.
- **출근 기록**: 동일한 근로자가 같은 날짜에 여러 번 저장될 경우, 내역 리스트에 중복으로 노출되는 현상 발생.

## 3. 목표
- 전 근로자(가족, 외국인, 아르바이트) 대상 **완벽한 CRUD(추가/조회/수정/삭제)** 기능 구현.
- 출근 내역 리스트에서 **중복 제거** (최신 기록 1건만 노출).
- 출근부 저장 시 기존 기록을 확실히 덮어쓰거나(Upsert) 중복 방지 로직 강화.

## 4. 작업 계획
1. **분석**: `WorkersPage`와 `AttendancePage` 코드 재검토 (완료).
2. **설계**: 수정 모달/폼 UI 설계 및 DB Upsert 로직 설계.
3. **구현 (Do)**:
    - `app/workers/page.tsx`: 근로자 정보 수정 기능 추가 및 UI 세분화.
    - `app/attendance/page.tsx`: 출근 기록 중복 방지 및 히스토리 리스트 정렬/중복 제거 로직 강화.
4. **검증 (Check)**: 갭 분석을 통한 기능 달성도 확인 (테스트 DB 연동).
5. **보고 (Act)**: 최종 보고 및 가이드 제공.
