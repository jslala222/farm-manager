# PDCA 계획서: 상세 페이지 데이터 가시성 확보 및 연동

## 1. 개해 (Plan - P)
현재 대시보드에는 요약 수치(수확량, 판매량 등)가 정상적으로 표시되나, 각 상세 메뉴(수확 기록, 판매 기록)에 진입했을 때 기존에 입력된 데이터 목록이 보이지 않아 사용자가 데이터 입력 유무를 확인하기 어려운 문제가 있습니다.

## 2. 현황 및 문제점
- **문제**: 수확 기록, 판매 기록 페이지가 '입력 폼'으로만 구성되어 있어 기존 내역 조회가 불가함.
- **원인 파악**: 초기 설계 시 모바일 입력 편의성에 집중하여 내역 리스트 기능을 포함하지 않음.

## 3. 목표
- 수확 기록, 판매 기록, 출근 체크 페이지 하단에 **'최근 5~10건의 내역'**을 보여주는 리스트 추가.
- 데이터를 입력한 직후 리스트가 실시간으로 갱신되도록 처리.
- 대시보드 수치와 상세 페이지 리스트의 데이터 정합성 보장.

## 4. 작업 계획
1. **분석**: 각 페이지의 현행 코드 구조 파악 (완료).
2. **설계**: 내역 리스트 UI 컴포넌트 설계 (모바일 최적화 카드 타입).
3. **구현 (Do)**: 
    - `app/harvest/page.tsx`에 최근 수확 내역 추가.
    - `app/sales/page.tsx`에 최근 판매 내역 추가.
    - `app/attendance/page.tsx`에 오늘의 출근자 명단 상세 표시 강화.
4. **검증 (Check)**: 입력 후 리스트 갱신 및 대시보드 합산 수치 일치 확인.
5. **보고 (Act)**: 작업 완료 보고 및 최종 사용자 가이드 제공.
