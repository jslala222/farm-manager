# PDCA 계획서 - 깃허브 업로드 (GitHub Push)

## 1. 개요 (Overview)
현재 개발 중인 딸기 농장 관리 프로그램(Strawberry Farm Manager)을 깃허브(GitHub) 원격 저장소에 업로드하여 버전 관리 및 협업이 가능하도록 설정합니다.

## 2. 목표 (Goals)
- 프로젝트 소스 코드를 깃허브에 안전하게 반영
- 민감한 정보(API Key, DB 접속 정보 등) 노출 방지
- 버전 관리 체계 구축

## 3. 작업 절차 (Plan)
1. **보안 점검 (Do)**
   - `.env` 파일 등 민감한 정보가 `.gitignore`에 포함되어 있는지 확인 (이미 확인됨)
   - 코드 내 하드코딩된 비밀번호나 키가 있는지 전수 점검
2. **저장소 설정 (Do)**
   - 깃허브에서 새로운 저장소(Repository) 생성 (사용자 수행 필요)
   - 로컬 프로젝트와 원격 저장소 연결 (`git remote add origin`)
3. **업로드 실행 (Do)**
   - 현재 변경 사항 커밋 확인
   - 원격 저장소로 푸시 (`git push -u origin master`)
4. **결과 확인 (Check)**
   - 깃허브 웹사이트에서 파일들이 정상적으로 올라갔는지 확인
5. **보고 및 마무리 (Act)**
   - 완료 보고 및 향후 관리 방안 제시

## 4. 기대 효과 (Expected Outcome)
- 코드의 안정적인 백업 및 관리
- 다른 환경에서도 개발 작업 지속 가능
