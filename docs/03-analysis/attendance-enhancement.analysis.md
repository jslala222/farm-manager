# [Analysis] 출근부 고도화 갭 분석 (Gap Analysis)

## 1. 요구사항 대비 구현 현황
| 요구사항 | 구현 상태 | 비고 |
|----------|-----------|------|
| 과거 날짜 선택 및 소급 저장 | 완료 | Date Picker 연동 및 handleSave 로직 고도화 |
| 아르바이트생 전용 가격/시간 입력 | 완료 | `part_time` 로직 분기 처리 완료 |
| 팀 단위(인원수) 출근 체크 | 완료 | `headcount` 필드 도입 및 통계 합계 반영 |
| 사장/가족/직원 전 직군 메모 기능 | 완료 | 모든 직군 이름 하단 메모창 노출 |
| 신규 가입자 소급 적용 | 완료 | DB `attendance_records` 스키마 확장 완료 |

## 2. 데이터 무결성 검토
- **일당 입력**: 숫자 외 문자 자동 제거 및 천단위 콤마 표시 기능 적용.
- **메모**: 공백 저장 허용(Nullable), 최대 길이 제한 없음.
- **날짜**: 선택된 날짜에 대해 기존 기록을 덮어쓰는(Delete-Insert) 방식으로 중복 방지.

## 3. 잔여 과제 (Gaps)
- **통계 고도화**: 현재는 최근 15개 기록 기반의 간이 대시보드만 제공함. 향후 월간 엑셀 다운로드 기능 등 추가 고려 필요.
