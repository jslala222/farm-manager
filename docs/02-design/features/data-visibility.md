# PDCA 설계서 (Design - D)

## 1. 개요
수확 및 판매 내역을 실시간으로 확인할 수 있는 상세 목록 인터페이스를 설계합니다.

## 2. 주요 설계 변경 사항

### 2.1 UI 컴포넌트 설계 (최근 내역 섹션)
- **위치**: 각 입력 페이지(수확, 판매) 하단에 배치.
- **스타일**: 모바일 친화적인 카드형 레이아웃 적용.
- **항목 구성**:
    - **수확**: 동 번호, 등급(스탬프 형태), 수량(박스), 기록 시간.
    - **판매**: 판매처(이름), 수량, 금액(원), 판매 유형(탭 아이콘).

### 2.2 클라이언트 데이터 페칭
- **초기 로드**: 페이지 진입 시 해당 농장의 최근 10건 내역 조회.
- **실시간 갱신**: 신규 데이터 저장(`handleSave`) 성공 시 `fetchHistory()` 함수를 재 호출하여 목록 즉시 갱신.
- **데이터 삭제**: 각 내역 옆에 '삭제(휴지통)' 버튼을 추가하여 실수로 입력한 데이터 삭제 기능 제공.
