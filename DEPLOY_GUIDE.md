# 🚀 자동 배포 가이드

## 빠른 시작

```bash
# 1단계: 변경사항 커밋
커밋

# 2단계: GitHub 푸시 + Vercel 배포
배포
```

> **💡 팁**: 두 번 실행하기만 하면 끝!

---

## 📦 커밋.cmd 사용법

### 목적
로컬 변경사항을 **자동으로 분석**하고 **의미있는 커밋 메시지**를 생성해 Git에 커밋

### 실행 방법
```bash
커밋
```

### 동작 과정
1. **변경 파일 감지** - 모든 수정/추가/삭제 파일 확인
2. **변경 내용 분석** - 코드 diff 분석
3. **커밋 메시지 생성** - 한글 + 이모지 포함 자동 생성
4. **Git 커밋** - 변경사항 로컬 저장

### 예시 커밋 메시지
```
feat: 비활성화 항목 필터 + 판매기록 안전 보호

- ⏸️ 비활성화 상태 UI 개선
- 🔒 판매기록 있으면 삭제 금지
- 📊 데이터 무결성 강화
```

---

## 🌍 배포.cmd 사용법

### 목적
커밋된 변경사항을 **GitHub에 푸시** → **Vercel 자동 배포** 트리거

### 실행 방법
```bash
배포
```

### 동작 과정
1. **사전 체크** - 커밋되지 않은 변경 있으면 중단
2. **Git 푸시** - `origin/main`으로 푸시
3. **Vercel 배포** - Git 연동으로 자동 배포 시작

### 주의사항
- ⚠️ **반드시 `커밋` 실행 후 사용!**
- 커밋되지 않은 변경이 있으면 배포 불가
- 배포 완료까지 3~5분 소요

## 🐛 문제 해결

### 배포.cmd 실행 중 오류

#### 오류: "ERROR: 커밋되지 않은 변경사항이 있습니다"
```bash
# 해결책: 로컬 변경사항 먼저 커밋
커밋
배포
```

#### 오류: "ERROR: git push 실패"
```bash
# 원인: 네트워크 문제 또는 GitHub 인증 문제
# 해결책: 
# 1) 네트워크 재확인
# 2) GitHub 인증 토큰 재설정
# 3) 수동으로 푸시 시도:
git push -u origin main
```

#### 오류: "ERROR: 이 폴더는 Git 저장소가 아닙니다"
```bash
# farm-manager 루트 폴더에서 실행 확인
cd c:\Users\User\Desktop\제미나이\ 3\연습\claude\projects_001\farm-manager
배포
```

## 💾 변경사항 저장 흐름

```
로컬 파일 수정
    ↓
git add (커밋.cmd가 자동 처리)
    ↓
git commit -m "메시지" (커밋.cmd 생성)
    ↓
local main 브랜치에 저장
    ↓
배포.cmd 실행
    ↓
git push origin main
    ↓
GitHub 저장소 업데이트
    ↓
Vercel 자동 감지 (Git 웹훅)
    ↓
Vercel 배포 시작
    ↓
배포 완료 (프로덕션 업데이트)
```

## 🎯 팁 & 베스트 프랙티스

### ✨ 권장 사항
1. **작은 단위로 커밋** - 한 번에 하나의 기능 커밋
2. **의미있는 작업 단위** - 관련된 변경사항만 함께 커밋
3. **배포 전 로컬 테스트** - 프로덕션 배포 전 확인 필수
4. **정기적인 백업** - 중요한 기능 구현 후 바로 커밋

### ⚠️ 피해야 할 것
- ❌ 여러 기능을 한 번에 커밋
- ❌ 테스트 안 하고 배포
- ❌ 미완성 코드를 프로덕션에 배포
- ❌ `git push -f` (강제 푸시) - 팀 작업 시 주의
