2안+ 구현 범위 분리안

즉시 구현 / 보류 / 실험안

이 문서는 지금까지 정리한 2안+ 기준을 실제 개발 범위로 나눈 것입니다. 핵심은 현장 사용성을 살리면서, 너무 무거운 기능은 뒤로 미루고, 구조만 먼저 열어두는 것입니다.

즉시 구현: 지금 바로 만들어도 운영에 도움이 되고 복잡도가 감당 가능한 항목
보류: 언젠가 필요하지만 지금 넣으면 무거워지는 항목
실험안: 구조는 열어두되 실제 사용 반응을 보고 확대 여부를 결정할 항목

1. 즉시 구현 범위

기능 설명 이유 위험도
현재재고 메인 화면 원물/가공품 현재 수량, 부족 상태, 기본 필터 사용자가 한눈에 봐야 재고관리가 시작됨 낮음
직접조정 기본형 폐기, 가공전환, 수동보정 현실 운영 변수를 가장 잘 반영함 낮음
조정이력 저장 누가, 언제, 무엇을, 얼마나, 왜 현재재고만 바꾸면 나중에 원인 추적이 안 됨 낮음
원물 판매 자동 차감 납품/택배 저장 시 원물 재고 차감 이미 기존 흐름과 연결 가능 중간
가공품 독립 재고 원물과 무관한 가공품도 재고 관리 전체 사용자 관점에서 필수 유연성 중간
재고부족 경고 모달 유지 기존 shortage dialog를 현재재고 기준으로 유지 지금까지 작업을 그대로 살릴 수 있음 낮음

2. 실험안

기능 설명 왜 실험안인가 권장 시점
등급변경 UI 특/상 → 중, 중 → 하 기록 현실성은 높지만 입력 부담이 생길 수 있음 2차
원물 → 가공전환 선택 연결 원물 차감과 가공품 증가를 느슨하게 연결 농가별 운영 차이가 커서 반응을 봐야 함 2차
입력자 추적 강화 직원별 조정 이력 강조 조직에 따라 필요도가 다름 2차
요약 리포트 폐기/가공전환/판매 현황 카드 좋지만 운영 핵심은 아님 2차~3차

3. 보류 항목

기능 설명 보류 이유
레시피 자동 환산 원물 몇 kg → 잼 몇 병 자동 계산 품목마다 규칙이 달라 초기 복잡도가 너무 큼
원물-가공품 완전 자동 연동 원물 차감과 가공품 생산을 완전 자동 처리 현재 단계에서는 과도하게 무거움
정밀 생산관리 가공 생산 배치, 원료 lot 추적 ERP 성격이 강해짐
창고/로케이션 관리 보관 위치별 재고 구분 운영 복잡도 급증
고급 분석 대시보드 폐기율, 회전율, 가공 효율 상세 분석 핵심 운영 안정화 후 고려하는 것이 맞음

4. 단계별 추천 로드맵

단계 내용 목표
1단계 현재재고 메인 + 직접조정 기본형 + 조정이력 재고관리의 기본 사용성 확보
2단계 원물/가공품 독립 재고 안정화 + 판매 자동 차감 정리 실운영 연결
3단계 등급변경, 선택 연결, 요약 리포트 실험 적용 현장 반응 확인
4단계 필요 시 원물-가공품 고급 연결 및 분석 확장 장기 확장
최종 판단: 지금은 즉시 구현 범위만으로도 충분히 의미 있는 재고관리 기능을 만들 수 있습니다. 특히 현재재고 + 직접조정 + 조정이력 조합이 가장 먼저 안정화되어야 합니다.
