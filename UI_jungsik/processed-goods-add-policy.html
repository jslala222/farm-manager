가공품 추가 정책

설정 등록형 vs 즉시 추가형

가공품은 계속 판매하는 정식 상품도 있고, 이번 몇 번만 파는 일회성 상품도 있습니다. 따라서 제품은 설정 등록형과 즉시 추가형을 모두 허용하는 쪽이 맞습니다.

1. 두 가지 추가 방식 비교

방식 설명 적합한 경우 권장 여부
설정 등록형 설정 화면에서 가공품을 미리 등록해두고 사용 계속 판매하는 정식 상품 필수
즉시 추가형 판매 또는 재고 화면에서 바로 상품을 생성 일회성 상품, 테스트 상품, 한정 상품 필수

2. 왜 즉시 추가형이 필요한가

상황 설명
이번 몇 번만 파는 상품 정식 등록까지 하기엔 번거롭지만 바로 팔아야 하는 경우
행사 / 시즌 한정 상품 짧은 기간만 판매하고 끝나는 상품
외부 원재료 가공품 원물과 무관하게 만들어 파는 상품
테스트 판매 반응을 보기 위해 잠깐 운영하는 상품

3. 품목 상태 구분

상태 설명 예시
정식 품목 설정에 등록되어 계속 사용하는 가공품 딸기잼, 딸기주스
임시 품목 즉시 추가로 생성된 일회성 또는 단기 상품 설 선물세트, 행사 한정청

4. 즉시 추가 시 최소 입력 항목

항목 필수 여부 설명
상품명 필수 사용자가 직접 입력
단위 필수 병 / 개 / 박스 / 세트
초기 수량 권장 즉시 재고에 반영할 시작 수량
품목 상태 자동 기본값은 `임시 품목`
원물 연결 여부 선택 필수로 묻지 않아도 됨

5. 어디서 즉시 추가할 것인가

화면 권장 여부 이유
판매 화면 권장 팔아야 하는 순간 바로 추가가 필요함
재고 화면 권장 재고를 먼저 잡고 판매할 수 있음
설정 화면 유지 정식 품목 등록용으로 필요

6. 데이터 품질을 위한 규칙

문제 예시 대응
이름 중복 딸기잼 / 딸기 잼 / 딸기쨈 유사 이름 추천 또는 기존 품목 검색 제공
단위 혼선 같은 상품을 병/개로 혼용 한 품목 기본 단위를 저장하고 재사용
임시 품목 누적 한 번 쓴 상품이 계속 목록에 남음 비활성 / 숨김 / 정식 전환 기능 추가

7. 1차 구현 권장안

항목 권장 판단 비고
설정 등록형 도입 정식 품목용
즉시 추가형 도입 임시 품목 생성 허용
임시 → 정식 전환 후속 1차 필수는 아님
원물 자동 연결 후속 강제하지 않음
결론: 가공품은 설정 등록형 + 즉시 추가형을 함께 가져가야 합니다. 특히 이번 몇 번만 파는 상품을 위해 즉시 추가 기능이 꼭 필요하며, 이때 생성된 품목은 임시 품목으로 취급하는 방식이 가장 현실적입니다.
