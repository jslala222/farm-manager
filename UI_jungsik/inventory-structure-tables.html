재고관리 구조 정리표

기준 방향은 2안+입니다. 즉, 현재재고 + 조정이력 분리형을 기본으로 하되, 3안의 일부인 가공전환 / 부분폐기 / 등급변경을 선반영 가능한 구조로 정리합니다.

기본 원칙: 원물 재고와 가공품 재고는 기본 분리가 맞습니다.
선택 연결: 필요할 때만 원물 → 가공전환을 연결합니다.
운영 관점: 사용자는 한눈에 보고 직접 수정할 수 있어야 하고, 출하/판매는 그 재고를 기준으로 자동 차감됩니다.

1. 원물 재고와 가공품 재고의 관계

항목 원물 재고 가공품 재고 설계 판단
재고 성격 수확으로 유입되는 농산물 판매용 완제품 또는 가공 상품 기본 분리
입고 방식 수확 입력 직접 생산등록 또는 수동 입고 기본 분리
출고 방식 납품/택배/폐기/가공전환 납품/택배/폐기 기본 분리
원물 연결 여부 자체 재고 연결될 수도 있고 아닐 수도 있음 선택 연결
예시 딸기, 고구마, 샤인머스켓 딸기잼, 즙, 외부원료 가공품, 선물세트 유연 설계 필요

2. 언제 분리하고 언제 연결하는가

상황 권장 처리 이유
사용자가 원물과 무관한 가공품을 판매함 가공품 재고를 독립 관리 원물 재고와 억지로 연결하면 운영과 안 맞음
사용자가 자기 원물 일부를 가공용으로 돌림 원물 → 가공전환 기록 원물 차감의 근거가 생기고 이력도 남음
가공품은 있지만 원물 소모량까지 아직 추적 안 함 가공품 독립 재고만 운영 처음부터 레시피/환산 추적을 안 해도 됨
장기적으로 원물-가공품 흐름 분석 필요 나중에 선택 연결 확장 2안 기반을 유지하면서 3안 일부만 확대 가능

3. 2안+ 화면 구조

화면 핵심 내용 사용자 행동
현재재고 화면 원물/가공품 현재 수량, 등급별 수량, 주의 품목, 부족 상태 한눈에 확인
직접조정 화면 폐기, 가공전환, 등급변경, 수동보정 수량과 이유 입력 후 재고 수정
조정이력 화면 언제, 누가, 무엇을, 얼마나, 왜 바꿨는지 추적 / 검토
납품/택배 화면 현재 재고 기준 자동 차감, 부족 시 경고 모달 출하 / 판매 저장

4. 조정 이벤트 표준안

이벤트 의미 예시 즉시 도입 여부
`manual_adjust` 수량을 직접 맞춤 실물 재고와 맞추기 위해 +5 / -3 즉시
`waste_out` 폐기 차감 특/상 2박스 폐기, 하 5kg 폐기 즉시
`process_out` 가공용 전환 딸기 원물 10kg → 가공용 차감 즉시
`grade_change` 등급 상태 이동 특/상 6박스 → 중 6박스 선반영
`sale_out` 판매/출하 차감 택배 4박스, 납품 10kg 이미 연결 중

5. 즉시 구현 / 보류 / 실험안

구분 항목
즉시 구현 현재재고 화면, 직접조정, 조정이력, 폐기, 가공전환, 수동보정, 판매 자동 차감
보류 레시피 자동 환산, 원물-가공품 완전 자동 연동, 정밀 생산관리
실험안 등급변경 이력, 가공품 원물 연결 옵션, 요약 리포트
판단 설명
왜 2안+인가 운영은 가볍게 가되, 데이터는 3안 일부를 담을 수 있기 때문
왜 기본 분리인가 모든 사용자가 원물 기반 가공품만 파는 것이 아니기 때문
왜 선택 연결인가 필요한 농가만 원물 → 가공전환을 쓰면 되기 때문
결론: 원물 재고와 가공품 재고는 기본 분리가 맞고, 필요 시에만 선택 연결합니다. 구현은 2안+로 시작해 현재재고/직접조정/조정이력/자동차감을 먼저 안정화하는 순서가 가장 안전합니다.
