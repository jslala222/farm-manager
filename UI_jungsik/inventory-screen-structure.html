2안+ 실제 화면 구조안

재고관리 화면 구성 문서

기준은 현재재고 + 조정이력 분리형입니다. 사용자는 한눈에 보고, 필요한 순간에만 직접 조정하고, 납품/택배는 그 재고를 기준으로 자동 차감되는 흐름을 목표로 합니다.

핵심 목표: 복잡한 관리자용 시스템이 아니라, 사용자/직원이 바로 쓸 수 있는 재고 화면
화면 원칙: 메인은 현재재고 중심, 조정은 최소 단계, 이력은 뒤에서 추적
구조 원칙: 원물과 가공품은 기본 분리, 필요 시 선택 연결

1. 전체 화면 맵

화면 목적 누가 주로 쓰는가 우선순위
현재재고 메인 현재 남은 재고를 한눈에 확인 사용자 / 직원 최우선
직접조정 폐기, 가공전환, 보정, 등급변경 사용자 / 직원 최우선
조정이력 왜 재고가 바뀌었는지 추적 사용자 / 관리자 높음
가공품 재고 독립 가공품 재고 조회 및 수정 사용자 / 직원 높음
요약 리포트 폐기, 가공전환, 판매 추이 확인 관리자 / 검토자 후순위

2. 현재재고 메인 화면

영역 들어갈 항목 설명
상단 요약 카드 총 품목 수, 부족 품목, 오늘 폐기, 오늘 가공전환 상태를 한눈에 파악
탭 / 필터 전체, 원물, 가공품, 부족만 보기 빠른 탐색
재고 목록 품목명, 단위, 현재수량, 등급별 수량, 상태 핵심 정보 집중
빠른 액션 폐기, 가공전환, 보정, 상세보기 목록에서 바로 조정 진입
경고 표시 부족, 음수, 주의 판매 전 미리 파악

3. 직접조정 화면

입력 항목 예시 비고
품목 선택 딸기 / 고구마 / 딸기잼 원물/가공품 구분 표시
조정 유형 폐기 / 가공전환 / 등급변경 / 수동보정 2안+ 핵심
현재 등급 특/상 / 중 / 하 원물만 해당 가능
변경 등급 특/상 → 중 등급변경 시만 표시
수량 5 박스 / 12 kg 모바일 숫자 입력 최적화
사유 무름 / 가공용 사용 / 실물차이 최소 이력 확보
저장 후 처리 현재 재고 반영 + 조정이력 저장 핵심 동작

4. 조정이력 화면

컬럼 예시 용도
일시 2026-03-07 19:30 언제 바뀌었는지
품목 딸기 무엇이 바뀌었는지
유형 폐기 / 가공전환 / 보정 / 등급변경 왜 바뀌었는지 분류
변경 내용 특/상 5박스 → 중 5박스 상세 확인
수량 5 박스 변화량
사유 보관 중 무름 나중 추적
작성자 직원A 책임 추적

5. 가공품 재고 화면

구성 내용
독립 가공품 목록 원물과 무관하게 관리되는 가공품
현재 수량 병, 박스, 개수 등 단위별 관리
직접 조정 폐기 / 수동보정 / 입고추가
판매 차감 택배/납품 시 자동 차감
선택 연결 요소 설명
원물 → 가공전환 필요한 사용자만 사용
가공품 생성 이력 원물과 연결할 경우 근거 저장
필수 여부 아님. 기본은 독립 가공품도 허용
추천 시점 2차 또는 실험안

6. 납품 / 택배 연동 포인트

상황 동작 메모
원물 판매 저장 현재 원물 재고 자동 차감 부족 시 기존 shortage dialog 사용
가공품 판매 저장 현재 가공품 재고 자동 차감 원물과 독립 가능
재고 부족 경고 / 차단 설정에 따라 동작 현재 구현 유지 가능
조정 후 재판매 조정된 현재 재고 기준으로 다시 차감 2안의 핵심 가치

7. 구현 우선순위 제안

단계 구현 내용 판단
1차 현재재고 메인 + 조정이력 + 직접조정 기본형 즉시
2차 폐기 / 가공전환 / 수동보정 안정화 즉시
3차 등급변경 UI 및 이력 실험안
4차 원물-가공품 선택 연결 실험안
5차 리포트 / 분석 / 고급 추적 후속
추천: 먼저 현재재고 화면과 직접조정을 고정하고, 그 다음에 조정이력을 붙이는 순서가 가장 안전합니다. 원물-가공품 선택 연결은 구조만 열어두고, 실제 도입은 후순위로 두는 것이 좋습니다.
