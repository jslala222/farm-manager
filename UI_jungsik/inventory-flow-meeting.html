비키트 회의용 정리본

재고관리 설계 3안 + 플로우차트 종합 문서

이번 논의의 관점은 관리자용 복잡한 ERP가 아니라, 사용자/직원 직접입력에 맞는 재고관리입니다.

핵심은 선별 여부도 현장 판단이며, 사용자가 한눈에 보고 직접 재고를 변경할 수 있어야 한다는 점입니다. 그렇게 정리되면 납품/택배 출하는 그 재고를 기준으로 자연스럽게 자동 차감됩니다.

1. 이번 회의에서 먼저 고정할 관점

관점 1. 재고의 선별 유무는 관리자가 아니라 사용자/직원 현장 판단에 달려 있다.
관점 2. 수확 시 등급은 참고 정보일 뿐, 이후 보관·재선별·가공·폐기로 충분히 바뀔 수 있다.
관점 3. 모든 등급(`특/상`, `중`, `하`)은 판매 / 가공 / 부분폐기 / 전부폐기로 갈 수 있다.
관점 4. 사용자는 복잡한 추적보다 현재 재고를 한눈에 보고 직접 수정할 수 있어야 한다.
관점 5. 출하/판매는 별도의 거대한 화면보다 재고 기반 자동 차감 흐름으로 붙는 것이 자연스럽다.

2. 추천하는 기본 플로우차트

핵심은 “등급 중심 고정 모델”이 아니라 “현재 재고를 직접 보고 조정하는 모델”입니다.

STEP 1

수확 입력

수확량을 입력합니다. 등급은 입력할 수도 있고, 안 할 수도 있습니다.

  • 원물 입고
  • 초기 수량 확보
  • 등급은 선택 정보
STEP 2

보관 / 재선별

현장 판단으로 재고 상태를 직접 변경합니다.

  • 특/상 → 중
  • 중 → 하
  • 등급 유지
STEP 3

재고 직접 조정

사용자/직원이 화면에서 현재 재고를 직접 바꿉니다.

  • 부분 폐기
  • 전부 폐기
  • 가공용 전환
  • 수동 보정
STEP 4

출하 / 판매

현재 재고를 기준으로 납품/택배가 자동 차감됩니다.

  • 원물 판매 출고
  • 가공품 판매 출고
  • 부족 시 경고
STEP 5

이력 / 리포트

나중에 무엇이 판매됐고, 무엇이 폐기됐고, 무엇이 가공으로 넘어갔는지 봅니다.

  • 판매 이력
  • 폐기 이력
  • 가공 전환 이력

3. 비키트 회의용 3안 제시

1안 · 가장 단순

원물 현재재고 직접수정형

수확은 입고로만 보고, 이후 현재 재고는 사용자가 직접 수정합니다. 출하/판매는 현재 재고에서 자동 차감됩니다.

플로우
수확 입력 → 현재 원물 재고 생성 → 사용자 직접 수정(등급/폐기/보정) → 납품/택배 자동 차감
장점: 가장 단순하고 현장 이해가 쉽습니다.
장점: 관리자 없이 사용자/직원이 바로 처리 가능합니다.
단점: 왜 재고가 바뀌었는지 이력이 약할 수 있습니다.
적합: 재고를 빨리 안정화하고 싶은 현재 단계
2안 · 균형형 / 추천

현재재고 + 조정이력 분리형

현재 재고는 한눈에 보이고 직접 수정 가능하게 하되, 수정 이유를 최소한의 이력으로 남깁니다.

플로우
수확 입력 → 현재 원물 재고 생성 → 사용자 직접 조정(폐기/가공전환/등급변경/보정) → 조정이력 저장 → 납품/택배 자동 차감
장점: 한눈에 보이면서도, 나중에 왜 바뀌었는지 추적이 가능합니다.
장점: 부분폐기, 전부폐기, 가공전환을 모두 담기 좋습니다.
단점: 1안보다 입력이 한 단계 늘어납니다.
적합: 실제 운영성과 데이터 신뢰도를 같이 가져가고 싶은 경우
3안 · 확장형

원물 / 가공 / 폐기 흐름 추적형

원물 재고에서 가공 전환, 폐기, 판매까지의 흐름을 모두 구조화해서 관리합니다.

플로우
수확 입력 → 선별/보관 → 등급변경 → 가공전환/폐기/판매 분기 → 각 이력 저장 → 리포트 분석
장점: 가장 정밀하고 미래 확장성이 큽니다.
장점: 가공품 원재료 추적까지 연결하기 좋습니다.
단점: 처음부터 구현하면 매우 무겁고 현장 입력 부담이 큽니다.
적합: 장기적으로 정밀 관리가 꼭 필요한 경우

4. 3안 비교표

비교 항목 1안 2안 3안
사용자 입력 난이도 낮음 중간 높음
한눈에 보기 매우 좋음 좋음 보통
직접 수정 편의성 매우 좋음 좋음 보통
부분폐기/전부폐기 반영 가능 가능 + 이력 보존 가능 + 세부 흐름 추적
가공 전환 반영 단순 처리 명확히 반영 가능 정밀 추적 가능
현장 도입 부담 낮음 낮음~중간 높음

5. 비키트 추천안

추천: 2안 `현재재고 + 조정이력 분리형`

왜냐하면 지금 관점은 “직접입력”과 “한눈에 보기”인데, 동시에 부분폐기·전부폐기·가공전환까지 담아야 하기 때문입니다.

이유 1. 현재 재고는 한눈에 보이고 직접 수정 가능합니다.
이유 2. 수정 이유를 최소 이력으로 남길 수 있어 데이터가 안 사라집니다.
이유 3. 원물에서 가공 전환, 일부 폐기 같은 현실 변수를 담기 좋습니다.
이유 4. 출하/판매는 그 재고를 기준으로 자동 차감 연결이 자연스럽습니다.

추천 구현 순서

단계내용
1차현재 재고 페이지 + 직접 수정 기능
2차폐기 / 가공전환 / 등급변경 조정이력 추가
3차출하/판매 자동 차감 안정화
4차가공품 연동 / 리포트 확장
핵심 결론: 재고관리는 처음부터 완전체로 만들기보다, 현재 재고를 한눈에 보고 직접 조정하는 구조로 시작하는 것이 현실적입니다. 그 위에 조정 이력을 얹고, 마지막에 출하/판매 자동 차감을 안정화하는 순서가 가장 안전합니다.
